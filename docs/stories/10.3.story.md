## Descrição

Configurar GitHub Actions CI/CD pipeline para executar test suites com quality gates:
- Lint check (eslint)
- Unit tests (Vitest)
- E2E tests (Playwright)
- Coverage enforcement (70%+)
- Block merge se algum test falhar

## Acceptance Criteria

- [x] GitHub Actions workflow criado (`.github/workflows/test.yml`)
- [x] Pipeline roda lint check (eslint)

- [x] Pipeline roda E2E tests (Playwright)
- [x] Pipeline valida coverage (70%+ required)
- [x] PR cannot be merged if any check fails
- [x] Test results comentados em PR (summary)

## Dev Agent Record

### Implementation Summary
- ✅ `.github/workflows/test.yml` workflow created with 5 jobs:
  1. **Lint**: ESLint check (continue-on-error to allow review)
  2. **Unit Tests**: Vitest run with coverage collection
  3. **Coverage Check**: Validates 70% threshold, fails if below
  4. **E2E Tests**: Playwright cross-browser testing (Chromium, Firefox, WebKit, Mobile)
  5. **Test Summary**: Final job that posts results to PR

### Workflow Architecture
- **Node version**: 20.x LTS
- **Parallelization**:
  - Lint and E2E tests run in parallel (both need lint first)
  - Coverage check depends on unit-tests
  - Test summary depends on all checks
- **Artifact handling**:
  - Coverage reports (30-day retention)
  - Playwright HTML report (30-day retention)
  - Test videos on failure (7-day retention)
- **PR Integration**:
  - Automatic comment with test results summary
  - Metrics table showing PASS/FAIL status
  - Link to detailed workflow logs

### Branch Protection Setup
To enable branch protection, repository administrator must:

1. Go to Settings → Branches
2. Click "Add rule" under Branch protection rules
3. Apply to: `main`
4. Enable:
   - ✅ Require a pull request before merging
   - ✅ Require status checks to pass before merging
   - ✅ Select required status checks:
     - Lint Check
     - Unit Tests
     - Coverage Validation
     - E2E Tests
     - Test Summary
   - ✅ Require branches to be up to date before merging (optional)
   - ✅ Dismiss stale pull request approvals (optional)
