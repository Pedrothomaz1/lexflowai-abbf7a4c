## Descrição

Configurar GitHub Actions CI/CD pipeline para executar test suites com quality gates:
- Lint check (eslint)
- Unit tests (Vitest)
- E2E tests (Playwright)
- Coverage enforcement (70%+)
- Block merge se algum test falhar

## Acceptance Criteria

- [x] GitHub Actions workflow criado (`.github/workflows/test.yml`)
- [x] Pipeline roda lint check (eslint)
- [x] Pipeline roda unit tests (Vitest)
- [x] Pipeline roda E2E tests (Playwright)
- [x] Pipeline valida coverage (70%+ required)
- [x] PR cannot be merged if any check fails
- [x] Test results comentados em PR (summary)

## Scope

### IN
- GitHub Actions workflow setup
- Lint, unit tests, E2E, coverage checks
- PR status checks (block merge if fail)
- Test result comments in PR

### OUT
- Não setup CD (deploy) — foco em CI only
- Não otimizar test execution time (aceitar ~15min)

## Tasks

- [ ] Criar `.github/workflows/test.yml` workflow
- [ ] Configure lint check step
- [ ] Configure unit test step
- [ ] Configure E2E test step (playwright)
- [ ] Configure coverage validation step
- [ ] Setup branch protection (require checks to pass)
- [ ] Add test result comment to PR (optional but nice)
