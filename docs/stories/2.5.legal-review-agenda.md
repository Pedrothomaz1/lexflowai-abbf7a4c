## Checklist de Dados Sensíveis (PII)

### Whitelist de Campos Exportados (verificar contra AC 1)
- ✅ User profile: id, email, full_name, created_at
- ✅ Contratos: id, title, valor, data_inicio, data_fim, status
- ✅ Requisições: id, title, valor, budget, status, approver
- ✅ Activity logs: user_id, action, timestamp, resource
- ✅ Consent logs: user_id, action ("granted"|"revoked"), timestamp
- ⚠️ **Auditoria necessária:** Nenhum campo de senha/token no export?

### PII Leakage Risks
- [ ] Email no export? **SIM (necessário)** — user precisa portabilidade
- [ ] Nome completo? **SIM (necessário)**
- [ ] Dados de contrato com informações sensíveis? ⚠️ **Revisar caso-a-caso**
- [ ] Activity logs com ações sensíveis? ⚠️ **Mascarar apenas PII extrema?**
