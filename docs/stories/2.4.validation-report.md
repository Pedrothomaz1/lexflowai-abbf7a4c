## Should-Fix Issues (Quality Improvements)

### Issue 1: Missing Risks & Mitigations Section (Severity: MEDIUM)
**Finding:** Story lacks explicit risk documentation despite complex approval routing logic.

**Details:**
- Approval routing has multiple branches (by value, by department, combinations)
- State machine has 5 states with various transitions
- Notification failures could cascade silently
- Pattern inconsistency with STORY-2.1 (which has explicit risks)

**Specific Risks to Document:**
1. Routing logic complexity → risk of untested edge cases (e.g., conflicting rules)
2. Notification failures → should not block approval (AC6 mentions this, but risk not documented)
3. State machine brittleness → snapshot tests might be needed for state transitions
4. Test flakiness → async notification testing can be unreliable

**Recommendation:** Add Risks & Mitigations section.

**Action:** Update story file with risk table (optional but recommended for consistency).

### Issue 2: Approval Routing Edge Cases Clarity (Severity: LOW)
**Finding:** AC2 (approval routing) specifies "by value" and "by department" but doesn't cover combinations or conflicts.

**Details:**
- AC2 shows examples: "R$ 50k → director" and "IT department → IT manager"
- But what if both rules apply? Priority order?
- What if no matching rule?

**Recommendation:** Expand AC2 with edge case clarification or add to Dev Notes section.

**Example:** "Routing hierarchy: (1) department-specific rule if exists, (2) value-based rule, (3) default escalation to CFO"

**Action:** Optional enhancement for @dev clarity.
