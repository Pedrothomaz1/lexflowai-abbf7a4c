## Descrição

Implementar cobertura de testes para `src/pages/ComplianceLGPD.tsx`, cobrindo:
- LGPD data export (all user data in portable format)
- Data retention policy enforcement
- User deletion flows (anonymization vs full deletion)
- Consent tracking
- Audit logging of all compliance actions

### Contexto

Compliance é CRÍTICO para operações legais em Brasil — violações de LGPD resultam em:
- 🔴 Multas (até 2% de revenue, máximo R$ 50M)
- 🔴 Ações judiciais (class action)
- 🔴 Danos à reputação

**Impacto**: Sem testes, compliance failures podem passar para produção desapercebidas.

**Business Impact**: LGPD violations = legal liability + shutdown risk.

## Acceptance Criteria

### AC 1: Unit Tests para Data Export
- [x] Teste: LGPD data export inclui todos dados do usuário
  - Given: User solicita data export
  - When: Export é gerado
  - Then: JSON inclui: profile, contratos, requisições, activity logs, etc
- [x] Teste: Export é em formato portável (JSON)
  - Given: Data export gerado
  - When: Arquivo é inspecionado
  - Then: Formato é JSON válido (RFC 7158)
- [x] Teste: PII é não-redacted em export
  - Given: Data export gerado
  - When: Arquivo é aberto
  - Then: Email, nome, dados pessoais estão presentes (não masked)

**Coverage Target**: 90% de export logic

### AC 2: Unit Tests para Data Retention
- [x] Teste: Old data é deletado conforme policy
  - Given: Retention policy = 1 ano
  - When: Cron job roda
  - Then: Data com > 1 ano é deletado
- [x] Teste: Deletion é auditado
  - Given: Data é deletado
  - When: Audit log é verificado
  - Then: Entrada existe com timestamp, user_id, data deletado

**Coverage Target**: 85% de retention logic

### AC 3: Unit Tests para User Deletion
- [x] Teste: User deletion anonymizes data (não completo delete)
  - Given: User deletion solicitado
  - When: Processo roda
  - Then: Nome → "Usuário Deletado", Email → hash, mas contratos permanecem (anonymized)
- [x] Teste: Full deletion remove dados sensíveis
  - Given: User deletion com "remove_all_data" flag
  - When: Processo roda
  - Then: Todos dados deletados, audit log permanece

**Coverage Target**: 85% de deletion logic

### AC 4: Unit Tests para Consent Tracking
- [x] Teste: Consent é registrado com timestamp
  - Given: User concorda com LGPD
  - When: Consent é salvo
  - Then: Record em `consent_logs` com timestamp + user_id
- [x] Teste: Revoke consent funciona
  - Given: User anteriormente consentiu
  - When: Revoke é clicado
  - Then: Novo record com "revoked" é criado

**Coverage Target**: 80% de consent logic

### AC 5: Integration Tests para Audit Logging
- [x] Teste: Todos compliance actions são auditados
  - Given: Data export solicitado
  - When: Ação é executada
  - Then: Audit log entry é criado
- [x] Teste: Audit logs não podem ser deletados
  - Given: Audit log entry existe
  - When: User tenta deletar entry
  - Then: Erro "Audit logs são imutáveis"

**Coverage Target**: 80% de audit logic

### AC 6: Error Handling
- [x] Teste: Export failure notifica user
- [x] Teste: Retention failure é logged + alertado

**Coverage Target**: 75%

### AC 7: Testes Passando em CI/CD
- [x] Coverage >= 80%
- [x] All checks passing

**Coverage Target**: 80% line coverage

### AC 8: Documentation & Compliance Verification
- [x] README documentando cada test (for compliance audit)
- [x] LGPD requirements checklist em README
- [x] Link para LGPD articles (article 5, 6, 18, 19, etc)
