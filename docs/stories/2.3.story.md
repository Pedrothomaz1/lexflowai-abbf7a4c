## Descrição

Implementar cobertura de testes para `src/pages/ContratoDetalhes.tsx` (1191 linhas), cobrindo:
- Form validation (campos, masks, constraints)
- Workflow state transitions (rascunho → aprovado → assinado → vigente)
- Approval process (quem pode aprovar, notificações)
- Error handling (validation errors, concurrent updates)

### Contexto

ContratoDetalhes é a principal interface para gestão de contratos — bugs aqui impactam operações diretas (contratos podem ser salvos com dados inválidos, approvals podem falhar, etc).

**Impacto**: Sem testes, regressões em validação/workflow podem deixar contratos em estado inconsistente.

**Business Impact**: Contratos inválidos = renegociação futura + overhead operacional.

## Acceptance Criteria

### AC 1: Unit Tests para Form Validation
- [ ] Teste: Email validation
  - Given: Campo email vazio
  - When: Form é submetido
  - Then: Erro "Email obrigatório"
- [ ] Teste: Data validation
  - Given: data_fim < data_inicio
  - When: Form é submetido
  - Then: Erro "Data fim deve ser após data início"
- [ ] Teste: Monetary value validation
  - Given: Valor < 0
  - When: Form é submetido
  - Then: Erro "Valor deve ser positivo"
- [ ] Teste: Required fields validation
  - Given: Campo obrigatório vazio
  - When: Form é submetido
  - Then: Erro apropriado para cada campo

**Coverage Target**: 90% de form validation logic

### AC 2: Unit Tests para Workflow State Transitions
- [ ] Teste: Draft → Em Aprovação transition
  - Given: Contrato em rascunho
  - When: "Enviar para aprovação" é clicado
  - Then: Status muda para "em_aprovacao"
- [ ] Teste: Em Aprovação → Aprovado transition
  - Given: Contrato em aprovação
  - When: Approver clica "Aprovar"
  - Then: Status = "aprovado"
- [ ] Teste: Aprovado → Assinado transition
  - Given: Contrato aprovado
  - When: Assinatura é obtida
  - Then: Status = "assinado"
- [ ] Teste: Invalid transitions são bloqueadas
  - Given: Contrato vigente
  - When: Tentar mudar para rascunho
  - Then: Erro "Transição inválida"

**Coverage Target**: 85% de workflow logic

### AC 3: Unit Tests para Approval Process
- [ ] Teste: Apenas autorizados podem aprovar
  - Given: User com role "analyst"
  - When: Tentar clicar "Aprovar"
  - Then: Botão desabilitado + mensagem "Permissão negada"
- [ ] Teste: Approval notifica stakeholders
  - Given: Contrato aprovado
  - When: Approval é salvo
  - Then: Notificação é enviada (mocked)

**Coverage Target**: 80% de approval logic

### AC 4: Integration Tests para Concurrent Updates
- [ ] Teste: Concurrent edit é detectado
  - Given: 2 users editam mesmo contrato simultaneamente
  - When: Primeiro salva, segundo tenta salvar
  - Then: "Contrato foi alterado por outro usuário" error

**Coverage Target**: 75% de concurrency handling

### AC 5: Error Handling Tests
- [ ] Teste: Network error durante save
- [ ] Teste: Validation error display
- [ ] Teste: Required field highlight

**Coverage Target**: 85% de error paths

### AC 6: Snapshot Tests
- [ ] Teste: Form rendered correctly (snapshot)
- [ ] Teste: Error messages layout (snapshot)

**Coverage Target**: 80%

### AC 7: Testes Passando em CI/CD
- [ ] Coverage >= 75%
- [ ] All checks passing (lint, type, tests)

**Coverage Target**: 75% line coverage

### AC 8: Documentation
- [ ] README com exemplos de workflow testing
- [ ] Guide para snapshot updates
